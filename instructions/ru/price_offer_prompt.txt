
**Инструкция для модели (только русский блок):**

**«Анализировать компоненты компьютера из JSON, классифицировать тип сборки (например, игровая, рабочая станция), предоставлять заголовки и описания на русском, переводить детали компонентов и возвращать структурированный JSON-ответ. Сохранять корректное форматирование, включать оценки уверенности и следовать подробным инструкциям для описаний и обработки компонентов.»**

В этом запросе слова `product` и `component` являются синонимами и обозначают компонент для сборки компьютера.

## Кодировка текста ответа: `UTF-8`

### **Шаблон ответа (русский блок):**

```json
{
    "title": "<название сборки на русском, сгенерированное вами>",
    "description": "<описание сборки на русском, сгенерированное вами>",

    "components": [
      {
        "component_name": "<название компонента на русском, сгенерированное вами>",
        "component_description": "<описание компонента на русском, сгенерированное вами. Если не удается создать спецификацию — оставьте поле пустым>",
        "component_specification": "<спецификация компонента на русском, сгенерированная вами. Если не удается создать спецификацию — оставьте поле пустым>",
        "image_url":"<Оставляешь как есть>"
      }
    ]
}
```

Примеры:

1. ПЛОХО: "Настольный компьютер IVORY:"
Хорошо: "Настольнук копьютер"

2. Плохо: "Этот компьютер идеально подходит для игр и работы с графикой."
Хорошо: "Компьютер отлично справляется с современными играми и профессиональными задачами."

3. Плохо: "Представляем настольный компьютер IVORY, который является отличным выбором ..."
Хорошо: "Этот компьютер — отличный выбор для ..."


Формат ответа `json` — строго соблюдать.
